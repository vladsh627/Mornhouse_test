# Mornhouse - Product Analytics Test Task

## Завдання 1: Розрахунок сукупного доходу сегменту Trial

### 1. Очищення даних та виявлення інсайтів
Під час аналізу вхідного датасету було виявлено наступні особливості:
* **Структура**: Довідкова інформація (метадані) була об'єднана з транзакційними даними. Для аналізу залишено лише перші 4 колонки.
* **Логіка прапорця Trial**: Згідно з документацією, `is_trial = 0` означає підписку **з** тріалом (інверсована логіка).
* **Ціноутворення Trial**: Тріал є платним ($6.99), подальші ребіли коштують $29.99.

### 2. Припущення (Assumptions)
Для розрахунку метрики "Сукупний дохід" (Cumulative Revenue) були прийняті наступні продуктові припущення:
1. **Gross Revenue**: Розраховується "брудний" дохід (до вирахування комісій App Store / Google Play 15-30% та податків).
2. **Success Rate**: Вважаємо, що всі транзакції (First payment та заявлена кількість ребілів) були успішно списані.
3. **Refund Rate = 0%**: Відсутність чарджбеків та повернень.

### 3. Формула розрахунку
Для кожного користувача в сегменті `is_trial = 0` дохід рахувався за формулою:
`User Revenue = 6.99 + (subscription_renewal_amount * 29.99)`
Для кожного користувача в сегменті `is_trial = 1` дохід рахувався за формулою:
`User Revenue = 40 + (subscription_renewal_amount * 40)`

Сукупний дохід — це сума `User Revenue` всіх користувачів даного сегменту.

### 4. Інструкція із запуску
Для відтворення процесу очищення даних та розрахунку:
1. Встановіть залежності: `pip install -r requirements.txt`
2. Запустіть скрипт: `python scripts/data_preparation.py`
3. Очищений датасет буде збережено у папку `data` для подальшої візуалізації в Tableau.

**Результат Завдання 1:** Сукупний дохід склав $1,745,581.66.
